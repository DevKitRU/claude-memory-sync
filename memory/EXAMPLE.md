# Шаблон memory-файла

Каждая запись в памяти — отдельный `.md` файл с фронтматтером. Индекс всех файлов живёт в `MEMORY.md` (см. рядом).

## Типы памяти

| Тип | Имя файла | Что туда |
|---|---|---|
| `user` | `user_<topic>.md` | роль пользователя, предпочтения, контекст |
| `feedback` | `feedback_<topic>.md` | корректировки («не делай X»), подтверждения нетривиальных паттернов |
| `project` | `project_<name>.md` | активные инициативы, цели, deadlines, стейкхолдеры |
| `reference` | `reference_<system>.md` | пойнтеры на внешние системы (Linear, дашборды, API) — **без самих секретов** |

## Структура feedback/project записи

```markdown
---
name: Краткий заголовок
description: Одна строка — по чему будет решаться релевантность в будущих сессиях
type: feedback  # или project, user, reference
---

Коротко — сама суть (правило или факт).

**Why:** почему это важно (часто — какой-то прошлый инцидент или сильное предпочтение).

**How to apply:** когда и как это правило срабатывает в будущих сессиях.
```

## Пример реального feedback

```markdown
---
name: Интеграционные тесты не мокают БД
description: после факапа с миграцией — все интеграционные тесты идут на реальный Postgres
type: feedback
---

Интеграционные тесты должны ходить в реальную БД (testcontainers), а не в моки.

**Why:** в марте 2026 моки пропустили сломанную миграцию — в продакшене уронили
пользователей. Разница моков с реальной схемой была невидима.

**How to apply:** если пишешь интеграционный тест и видишь `jest.mock('db')` —
замени на testcontainers + реальный драйвер. Unit-тесты бизнес-логики моки могут
использовать.
```

## Что НЕ хранить

- Секреты (API-ключи, токены, пароли). Держи их в `~/.claude/secrets/` вне git-репо.
- Код-паттерны, архитектура, файловые пути — это всё выводится из самого кода.
- История коммитов — для этого есть `git log`.
- Debug-рецепты из вчерашнего бага — он уже в коммите, не засоряй память.

## Индекс MEMORY.md

Когда создаёшь новый memory-файл — добавь строчку в `MEMORY.md`:

```markdown
- [file_name.md](file_name.md) — краткое описание для поиска релевантности
```
